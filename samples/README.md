# Samples — example inputs + outputs

Bundled examples so you can see the data shapes without needing AWS or the full MIMIC dataset.

```
samples/
├── cxr-images/              # 5 preprocessed CXR PNGs (~130KB each, MIMIC-CXR derived)
├── phase2-cxr-results/      # Phase 2 prediction JSONs (10 samples) + test_mimic.jpg
├── mock-emr/
│   ├── worklist.json        # 30-patient demo worklist (shown on doctor's login page)
│   └── patients/            # 5 representative FHIR Bundle JSONs (~14KB each)
├── final-reports/           # 2 generated clinical report PDFs (45KB each)
└── reference-data/          # Bundled reference data used by Phase 3/4
    ├── lab_reference_ranges_v9_5.yaml      (340KB) — 145 lab items × 18 categories
    ├── lung_disease_profiles_v3_6.yaml     (2.3MB) — 49 disease YAML profiles
    ├── korean_hpo_dictionary_v1.json       (33KB)  — Korean translations
    └── icd10_reference_v1.json             (57KB)  — ICD-10 sub-code mappings
```

## Example workflows

### 1. Inspect a CXR prediction
```bash
cat samples/phase2-cxr-results/002d3bad91194209bc888d3b582d40df.json | jq .
```
Shows 14-class DenseNet output (Atelectasis, Cardiomegaly, ..., Pneumothorax) with HPO codes + `positive_hpos` list (labels > threshold 0.3).

### 2. View a generated clinical report
```bash
open samples/final-reports/2a2cfd35-b528-4c93-bf23-f18c1d52b58f/report.pdf
```
This is the same PDF format the Phase 5 RAG Lambda generates and stores at `s3://bucket/final_reports/{session_id}/report.pdf`.

### 3. Test pipeline locally with mock data
```bash
# Upload sample patient FHIR bundles to your S3:
aws s3 sync samples/mock-emr/ s3://$S3_BUCKET/mock-emr/ --sse AES256

# Run the API + frontend in dev, click any of the 5 demo patients
```

### 4. Generate a heatmap (Grad-CAM overlay on CXR)

The production Phase 2 Lambda returns **predictions only** — no heatmaps. To generate Grad-CAM visualizations:

```bash
cd lambdas/phase2-vision
python ../../scripts/local-inference-with-heatmap.py \
  --image samples/cxr-images/00094318-ffb6a7d8-6dd1e667-280271ad-ae663064.png \
  --checkpoint ./models/latest_checkpoint.pth \
  --out /tmp/heatmap_overlay.png
```

Implementation reference: `s3://say2-2team-bucket/Phase_2/soo_net_5.py` lines 220-235 — uses `cv2.applyColorMap(..., COLORMAP_JET)` + `cv2.addWeighted(original, 0.6, heatmap, 0.4, 0)`.

Heatmaps are useful for clinician visualization but currently not auto-generated by the pipeline. If you want to add this:
1. Modify `lambdas/phase2-vision/lambda/phase2_handler.py` to call the GradCAM hook after SageMaker invoke
2. Upload the overlay PNG to `s3://bucket/heatmaps/{session_id}.png`
3. Add `heatmap_s3_key` field to `phase2_xray_processing` table
4. Frontend renders `<img src={heatmapUrl}>` next to the prediction list
